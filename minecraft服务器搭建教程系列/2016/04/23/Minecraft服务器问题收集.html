Minecraft服务器问题收集

LongerONE 23 Apr 2016

本文将对运行Minecraft服务器所遇到的一些问题进行收集并适当说明原因，同时介绍解决方案。

问题收集

1. 打开启动批处理后提示：’java’不是内部或外部命令，也不是可运行的程序。

  • 原因：没有安装java运行环境或安装不正确；
  • 解决方案：从Java官网下载正确的java运行环境并安装。
    检查java是否成功安装：打开 运行（Win + R），输入 cmd 后回车，出现黑色命名窗口，输入 java -version 后回车，如果显示 java的版本信息则安装成功：

2. 服务器启动完成后提示：Can’t keep up! Did the system time change,or is the server overloaded? Runing 2052ms behind,skipping 41 tick(s).

Can't keep up! Did the system time change,or is the server overloaded? Runing 2052ms behind,skipping 41 tick(s).
  • 大致意思是：服务器运行时有延后，是不是系统时间改变或服务器过载？！
  • 原因：性能跟不上
  • 解决方案：尝试给服务器设置更大内存或升级硬件。

3. 进入服务器是提示：登入失败：无效的回话（请尝试重启你的游戏），后台提示：com.mojang.authlib.GameProfile@xxxxxx…


  • 原因：因服务器未关闭正版验证，导致进入时授权失败！
  • 解决方案：打开 server.properties，将 online-mode=true 改为 online-mode=false后保存，重启服务器生效。

4. 进入服务器时卡在：登入中…

情况一：

  • 原因：出现这种情况通常是因为服务端使用Cauldron，而客户端使用的是原版，导致服务端与客户端不兼容。
  • 解决方案：客户端启动时选择forge，即使没有安装任何mod：

情况二：

  • 原因：网络质量欠佳或加载地图时卡住
  • 解决方案：取消后重新进入服务器

5. Minecraft 1.8.9服务端安装神秘时代5后无法正常启动，控制台提示：

注意：安装神秘时代4可能出现同样问题。

  • 原因：神秘时代需要前置MOD:Bauble(饰品)，但服务器启动时下载失败！
  • 解决方案：手动将 Bauble mod放入 mods —> 1.8.9 文件夹内后启动服务器。

6.Error: Unable to access jarfile X:\Minecraft\Server\cauldron-1.7.10-1.1388.1.0-server

Error: Unable to access jarfile X:\Minecraft\Server\cauldron-1.7.10-1.1388.1.0-server
  • 既：不能读写jar文件：X:\Minecraft\Server\cauldron-1.7.10-1.1388.1.0-server
  • 原因：服务器核心文件损坏。
  • 解决方案：重新下载服务器核心文件：cauldron-1.7.10-1.1388.1.0-server.jar。

7.Error: Invalid or corrupt jarfile craftbukkit-1.9.2.jar

Error: Invalid or corrupt jarfile craftbukkit-1.9.2.jar
  • 既：无效或损坏的jar文件craftbukkit-1.9.2.jar
  • 原因：服务器核心文件损坏。
  • 解决方案：重新下载服务器核心文件 craftbukkit-1.9.2.jar.

8.Error occurred during initialization of VM Could not reserve enough sapce for object heap

Error occurred during initialization of VM Could not reserve enough sapce for object heap
  • 既：在虚拟机初始化时遇到错误，不能为项目分配足够的堆内存。
  • 原因：为 Minecraft 服务器设置的内存大于计算机的空闲内存。
  • 解决方案: * 将Minecraft服务器内存设置小一些
    * 或者升级计算机内存（加内存条）
