Minecraft1.7服务器搭建教程

LongerONE 18 Apr 2016

前言

本文将介绍 Minecraft 1.7 的服务器搭建方法，使用的服务端为Cauldron，可同时使用 Mod 和 插件。鉴于目前主流使用 1.7.10，故使用该版本进行时示范，1.7.2搭建方法类似。

目前 Cauldron已停止更新。

Cauldron 大锅，炼药锅

思路

整体思路为：使用 java 运行服务端核心文件，核心文件会自动读取并加载Mod和插件。

步骤

1.环境准备：

安装 java 运行环境。

2.文件准备：

Cauldron官方下载地址：Cauldron服务端资源下载

备用网盘：备用资源下载网盘 提取码： kknc

需要下载的文件：

  1. 核心文件 cauldron-1.7.10-1.1388.1.0-server.jar
  2. 与核心文件版本对应的运行所需相关文件：libraries-1.1388.1.0.zip

3.目录准备

新建一个文件夹，比如：Server1.7.10，将核心文件 cauldron-1.7.10-1.1388.1.0-server.jar 复制到该文件夹，将 libraries-1.1388.1.0.zip 解压到该文件夹，目录结构如下图：

4.新建启动文本

在上面的文件夹内新建一个 txt 文本文件，名为 启动服务器.txt（后面操作默认在该文件夹）；

5.输入启动命令

打开 启动服务器.txt，输入以下代码后保存：

java -jar -Xms512M -Xmx512M  cauldron-1.7.10-1.1388.1.0-server.jar
pause
  • 其中：cauldron-1.7.10-1.1388.1.0-server.jar 为核心文件的文件名，它将会使用同一目录下的minecraft_server.1.7.10.jar文件，所以请不要删除它们；
  • -Xmx 设置JVM最大可用内存；
  • -Xms 设置JVM促使内存；
  • 512M 为服务器内存大小，将-Xms -Xmx设置为相同大小，可避免JVM在完成垃圾回收后对内存进行重新分配，可在一定程度上提升服务器性能及稳定性;
  • pause 是为了在窗口试图关闭时能继续保留窗口，以查看相关信息。

6.文本改为批处理

将 启动服务器.txt 改为 启动服务器.bat

7.第一次运行

打开 启动服务器.bat ，会出现黑色CMD命令窗口，在窗口中会有如下提示:

  • 提示信息可能需要仔细查找；
  • 如果使用 kCauldron 服务端，可能会自动同意EULA并修改eula.txt，窗口将不会出现提示。

8.同意用户许可协议

使用 kCauldron 服务端跳过此步。

打开 eula.txt ，推荐使用 Notepad++ 打开，将 false 改为 true，然后保存：

9.添加Mod及插件：

将准备好的Mod放入 mods 文件夹，如下图：

将准备好的插件放入 “plugins”文件夹，如下图：

10.修改服务器配置

打开 server.properties 文件，推荐 Notepad++ ,将 online-mode=true 改为 online-mode=false，否则只能正版用户才能登录，然后保存.

11.第二次启动服务器

打开 启动服务器.bat ，将会出现黑色窗口，随后静静的等待，以下状态为正在生成地图： 以下状态为启动完毕，通常会提示：

Done

GroupManager - INFO - Bukkit Permissions Updated!

12.客户端连接

连接服务器的客户端需要与服务器有相同的版本及Mod(单机的mod除外，如：小地图):

  • 服务端必须与客户端有相同Mod，版本需一致；
  • 客户端独有，服务端可没有的 Mod举例:
    • 小地图
    • 各类信息显示Mod（如：伤害显示、游戏信息显示等）
    • NEI 及 扩展
    • BC的 BuildCompat
    • OptiFine高清修复
    • 提升性能mod（如：fastCraft）
    • 字体修复
    • 皮肤模组
    • 动态光源
    • 鼠标手势
    • R键整理
    • 中文输入修复
    • 光影模组

启动客户端后进入 多人游戏 —> 添加服务器，地址输入 127.0.0.1 （本机）：

  • ip地址 127.0.0.1 是指本机，同 localhost
  • 如果使用同一路由器，可使用路由器分配的IP，如：192.168.1.102
  • 如果是从外网接入，详细见网络相关教程。

以下状态为连接服务器成功后：

  • 如果服务端与客户版本不同，可能无法显示服务器状态。
