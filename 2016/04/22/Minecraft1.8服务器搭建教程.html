Sponge 海绵

Sponge介绍

Minecraft服务器的搭建在1.8以前，使用水桶服Bukkit的较多，基于水桶服的插件也是鳞次梓比，而且能够使用Mod的服务端如：MCPC+,Cauldron都可以使用水桶服的插件。

不过情况目前已发生变化，由于诸多原因，Minecraft插件开发的主要力量都集中于开发一个叫做 Sponge 的新项目。而且这个新的Sponge将成为Minecraft1.8服务器的主流，目前Sponge仍只有公测版本。

Sponge将使用全新的插件机制，在使用方法上与Bukkit有所不同。

Sponge其实为一个Mod，一个可以使用插件的Mod.

文件准备注意事项

  • Sponge最好与Forge版本对应：

    • Forge 下的 1808 指的是Forge服务端的版本，故需要使用 1808 版本的Forge安装器。

4.新建启动批处理

新建一个文本文档，如：启动服务器.txt，打开输入以下代码后保存：

java -jar -Xms1G -Xmx1G forge-1.8.9-11.15.1.1808-universal.jar
PAUSE
  • 其中：forge-1.8.9-11.15.1.1808-universal.jar 为核心文件的文件名，它将会使用同一目录下的minecraft_server.1.8.9.jar文件，所以请不要删除它们；
  • -Xmx 设置JVM最大可用内存；
  • -Xms 设置JVM促使内存；
  • 1G 为服务器内存大小，将-Xms -Xmx设置为相同大小，可避免JVM在完成垃圾回收后对内存进行重新分配，可在一定程度上提升服务器性能及稳定性;
  • PAUSE 是为了在窗口试图关闭时能继续保留窗口，以查看相关信息。

将 启动服务器.txt 改为 启动服务器.bat

5.第一次启动

双击打开 启动服务器.bat，出现黑色CMD控制台窗口，稍等片刻后窗口内出现如下提示：

You need to agree to the EULA in order to run the server. Go to eula.txt for more info. 意思是: *你需要同意EULA(用户许可协议)才可以运行服务器，到 `eula.txt`查看更多信息。*  如图：   ![EULA](/images/mc18s_EULA.png "EULA")
